1. 文档概述
1.1 项目背景
当前医院医务管理系统核心制度指标运营存在两大核心痛点：
1）口径不统一：各院区、合作医院对医疗质量核心制度指标统计口径存在个性化差
异，国标标准口径无法直接复用，传统硬编码适配模式迭代慢、维护成本高；
2）异常排查黑盒化：指标数据异常时，无法快速区分根因是【系统字段缺失/表结构
变更】、【自定义口径规则错误/冲突】还是【业务原始脏数据】，依赖资深开发逐层
排查，效率极低。
为解决以上问题，拟建设医务核心制度指标 AI Agent 自助平台，基于系统表模型+结
构化指标规则知识库，实现指标自动生成、多口径差异化适配、全链路自助根因排
查，赋能业务自助配置、技术减负提效。
1.2 建设目标
1. 构建标准化+个性化双层指标规则知识库，支持每家医院独立定制统计口径，实现口
径可配置、可版本化、可追溯；
2. 实现 AI Agent 自动生成指标计算逻辑、可执行 SQL、指标元数据文档，替代人工
编写适配；
3. 建立三层智能排查引擎，精准区分【字段问题/口径问题/数据问题】三类异常根
因，输出标准化诊断报告与整改方案；
4. 实现指标配置、计算、校验、排查、复盘全流程自助化，减少研发重复对接与迭代
成本。
1.3 适用范围
本文档适用于医务管理系统核心制度指标模块 AI Agent 功能的需求评审、开发落地、
测试验收、项目立项，覆盖所有医院院区的核心制度指标（三级查房、术前讨论、危
急值管理、手术分级、院内感染、查对制度等）统计与运维场景。
2. 整体系统架构

2.1 四层技术架构
1. 底层资源层
包含数据库元数据、双层指标规则知识库、全量运算日志池、业务数据快照。支撑所

2. AI Agent 核心引擎层（核心自研能力）
由 5 个专用智能体协同工作，分工明确、解耦可迭代：元数据解析 Agent、指标生成
Agent、口径适配 Agent、故障根因排查 Agent、人机交互 Agent。
3. 业务应用层
集成至现有医务系统后台，包含口径配置管理、指标自助生成、异常一键排查、指标
运算监控、版本回溯管理等功能页面。
4. 输出交付层
输出可执行 SQL、标准化指标文档、异常诊断报告、缺失字段整改清单、口径差异对
比报表、指标波动预警信息。
2.2 核心能力逻辑闭环
元数据同步 → 知识库规则匹配 → AI 自动生成指标逻辑 → 口径分层运算校验 → 异常
智能分层排查 → 问题整改复盘 → 规则迭代入库
3. 功能需求详细说明
3.1 元数据解析 Agent 功能需求
核心作用：自动感知系统表模型，解决字段缺失、表结构变更导致的指标异常，为指
标生成和排查提供底层数据支撑。
功能 1：全量元数据自动同步
1. 定时扫描数据库 INFORMATION_SCHEMA，自动同步业务表、字段、字段类型、
注释、主键、外键关联、是否可为空、默认值；
2. 支持手动触发同步，适配临时表结构变更场景；
3. 自动识别医疗业务语义，匹配查房、手术、危急值、患者、医师、科室等业务实
体。
功能 2：表结构变更监控告警
1. 监测表新增、删除、字段改名、字段类型变更、字段删除行为；
2. 自动关联受影响的所有指标，推送变更告警；
3. 记录结构变更日志，支持回溯查询。

功能 3：字段可用性预校验
指标计算、生成前自动校验依赖表和字段是否存在、是否可用，提前拦截结构异常问
题。
3.2 双层指标规则知识库功能需求
彻底解决多医院口径不统一问题，采用国标只读基础库+医院定制私有库双层设计，
优先级：医院定制口径 > 国标标准口径。
1. 国标标准规则库（全局统一、不可删除）
1. 录入卫健委医疗质量安全 18 项核心制度标准指标口径；
2. 每条指标包含：指标名称、指标定义、统计周期、统计维度、分子计算规则、分母
计算规则、筛选条件、依赖业务表、校验阈值、业务释义；
3. 支持后台统一更新国标规则，更新后自动推送差异通知至各医院。
2. 医院个性化定制库（单院隔离、可编辑）
1. 各医院独立规则空间，数据隔离互不干扰；
2. 支持自定义修改：统计时间范围、人员范围、科室范围、特殊剔除规则、字段替
换、筛选条件叠加、分母分子特殊修正逻辑；
3. 支持口径启停、临时生效、定时生效配置。
3. 口径冲突校验能力
AI 自动校验自定义口径与国标口径的逻辑冲突、条件互斥、维度重叠、规则冗余问
题，提前预警并给出优化建议。
3.3 指标生成 Agent 功能需求
实现自然语言需求一键生成可落地指标，替代人工写 SQL、整理文档。
核心工作流程
1. 用户输入自然语言需求（例：XX 医院三级查房合格率，剔除周末急诊临时查房，仅
统计主治及以上医师）；
2. Agent 自动匹配国标模板 + 该院定制规则；
3. 自动校验数据库字段完整性，缺失则预警；
4. 自动生成：分层可执行 SQL（标注国标逻辑、定制逻辑）、指标元数据、口径说
明、依赖资源清单；
5. 支持一键保存、一键试运行、批量生成全套核心指标。
生成内容标准
1. SQL 代码分段注释，清晰区分标准口径与医院定制差异；
2. 自动规避分母为 0、时间区间异常、空值报错等常见问题；
3. 自动输出完整指标文档，可直接用于医务审核、项目交付。

3.4 故障根因排查 Agent（核心重点功能）
解决核心痛点：精准区分字段问题、口径问题、数据问题，实现无人自助排查。
三级分层排查机制（固定优先级）
第一层：系统结构校验——定位【字段/表模型问题】
1. 提取指标所有依赖表、字段；
2. 比对实时数据库元数据，判定表不存在、字段缺失、字段类型不匹配、字段废弃等
问题；
3. 输出：缺失字段清单、结构异常说明、适配修改方案。
第二层：口径规则校验——定位【定制口径/逻辑问题】
1. 分别独立执行【纯国标口径】、【医院定制口径】两套计算逻辑；
2. 通过结果差值、逻辑校验判定问题归属：国标正常、定制异常=口径规则问题；
3. 自动识别规则漏洞：筛选条件互斥、时间重叠、人员范围错误、逻辑冗余、分母为
0；
4. 输出口径差异对比、错误点位、修正 SQL。
第三层：原始数据校验——定位【业务脏数据问题】
1. 在结构、口径均无异常的前提下，校验原始业务明细数据；
2. 识别空值过多、时间逻辑矛盾（入院晚于出院、查房晚于手术）、重复数据、编码
不规范、无效记录；
3. 统计脏数据占比、导出异常明细、给出数据治理建议。
自助排查输出物
1. 标准化诊断结论（明确唯一或多重根因）；
2. 问题明细清单、异常样例；
3. 临时修复 SQL、长期整改方案；
4. 指标重跑验证入口。
3.5 口径版本与变更管理功能
1. 所有医院定制口径版本化管理，每次修改自动生成版本号、记录操作人、操作时
间、修改内容；
2. 支持版本回溯、历史口径对比、一键恢复历史版本；
3. 支持口径变更日志全追溯，满足医疗质量审计要求；
4. 支持批量同步口径规则至同类型医院。
3.6 指标运算与预警功能
1. 兼容现有定时任务框架，支持指标定时重算、手动重算、指定周期重算；
