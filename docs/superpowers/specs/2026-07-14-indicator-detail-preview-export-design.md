# 指标明细查看与短期导出设计

## 目标

在指标试运行得到分母、分子和未达标数量后，允许具备权限的医院人员查看对应业务明细，并生成可追溯、24小时自动过期的Excel文件。功能首先覆盖当前已经落地的四个指标，不把患者明细写入聊天记录、执行链路或Git仓库。

## 方案选择

采用“按需生成短期明细快照”：只有用户点击“查看详情”时才查询患者级明细。系统使用本次试运行保存的统计区间、医院、指标计算定义、字段映射和生效口径生成明细查询；查询完成后核对分母、分子数量，数量一致才允许生成Excel。

不采用以下方案：

1. 每次试运行自动保存全部明细：会在无人查看时也保存患者数据。
2. 长期建立指标明细库：会重复存储医院业务数据，增加同步、隔离和合规成本。

## 用户流程

### 登录

医院人员登录改为后端真实校验。本地演示账号为：

- 账号：`user_001`
- 初始密码：`123456`
- 医院：`hospital_001`
- 权限：`indicator_detail_view`、`indicator_detail_export`
- 首次登录必须修改密码；生产部署不得继续使用演示密码。

第一版使用本地账号表和权限表，后续接医院统一身份认证或单点登录时，继续使用相同的医院范围和权限码。

### 查看明细

试运行结果表增加“操作”列：

| 统计项 | 数量 | 操作 |
| --- | ---: | --- |
| 统计范围（分母） | 576 | 查看详情 |
| 达到要求（分子） | 488 | 查看详情 |
| 未达到要求 | 88 | 查看详情 |
| 指标结果 | 84.72% | - |

点击“查看详情”打开明细窗口，窗口包含三个标签：

1. **统计范围**：本次进入分母的全部记录。
2. **达到要求**：分母中满足分子条件的记录。
3. **未达到要求**：分母中不满足分子条件的记录。

默认每页显示50条脱敏数据，支持翻页。页面显示指标、本院口径、统计区间、快照时间和总条数。

### 生成Excel

点击“生成并下载Excel”前进行二次确认，明确文件包含患者级明细和24小时保留期限。生成后页面显示：

- 文件名
- 项目内相对路径
- 生成时间
- 到期时间
- 下载按钮

文件保存到：

```text
runtime/exports/{hospital_id}/{run_id}/{export_id}.xlsx
```

`runtime/` 已被Git忽略。浏览器不能直接打开服务器本地路径，因此页面使用脱敏预览查看内容；下载Excel后由操作系统中的Excel或兼容软件打开。

## 明细字段

禁止使用 `SELECT *`。结构化指标定义增加 `detail_fields`，只查询业务核对需要的记录标识和证据字段。

首批四个指标字段范围：

| 指标 | 明细字段 |
| --- | --- |
| 患者入院48小时内转科比例 | 入院流水号、入院时间、转科时间、转出科室、转入科室、转科耗时、是否达到要求 |
| 急会诊及时到位率 | 患者标识、科室、会诊类型、申请时间、到位时间、到位耗时、是否达到要求 |
| 急危重症患者抢救成功率 | 患者标识、抢救事件号、科室、抢救时间、严重程度、抢救结果、是否达到要求 |
| 术中自体血回输率 | 患者标识、手术号、科室、手术时间、术中输血标志、自体血回输标志、是否达到要求 |

字段必须经过本院已确认字段映射。缺少记录标识或证据字段时停止生成，提示实施人员先完成字段映射，不根据字段名猜测。

## 查询与一致性

### 运行快照

试运行日志增加以下非患者级快照信息：

- `numerator_count`
- `denominator_count`
- `run_context_json`

`run_context_json`只保存指标计算定义、字段映射、生效参数、口径版本、统计区间和数据库来源，不保存患者记录。

### 明细查询

新增确定性明细SQL生成器。它从运行快照构造一条只读查询：

1. `WHERE`使用与试运行分母相同的医院、统计区间和分母条件。
2. `SELECT`只包含 `detail_fields` 及必要派生值。
3. 使用确定性 `CASE WHEN` 标记每条记录是否进入分子。
4. 应用层将一次查询结果拆成分母、分子和未达标三个集合。

查询必须通过既有只读SQL安全校验，并强制医院范围、统计区间、超时和最大导出行数。第一版最大导出20,000条，超过时提示缩小统计区间，不截断后伪装成完整数据。

数量校验通过后，完整查询结果写入应用内部短期快照：

```text
runtime/exports/{hospital_id}/{run_id}/{snapshot_id}.jsonl.gz
```

快照文件不提供直接下载，不在接口中返回绝对路径。脱敏预览和Excel都从同一个快照读取，不再次查询业务库，从而保证用户看到的页面明细和下载文件一致。快照与Excel使用同一到期时间并一起清理。

### 数量校验

明细查询结束后重新计算数量，并与试运行日志核对：

- 数量一致：创建短期快照，允许预览和导出。
- 数量不一致：不生成文件，提示“业务数据已经变化，请重新试运行后查看明细”。

这保证Excel显示的分母、分子数量与页面指标值一致。明细快照记录自身生成时间，不声称代表数据库之后发生的变化。

## Excel结构

使用 `.xlsx` 格式，一个文件包含三个工作表：

1. `统计范围_576`
2. `达到要求_488`
3. `未达到要求_88`

每个工作表顶部包含指标名称、医院、本院口径、统计区间、导出人、导出时间和数据说明；下方为明细表格。文件中的患者标识保留完整值，页面预览使用脱敏值。

Excel公式不包含可执行宏，不写外部链接，不把原始SQL写入文件。

## 权限与认证

### 本地账号

新增本地用户、权限和会话表。密码使用带随机盐的PBKDF2-HMAC-SHA256保存，不保存明文密码。会话使用随机令牌，数据库只保存令牌摘要，默认8小时过期。

医院登录成功后，前端在 `sessionStorage` 保存令牌并为明细接口附加 `Authorization: Bearer <token>`。退出登录立即撤销会话。

### 权限边界

- `indicator_detail_view`：允许查看脱敏预览。
- `indicator_detail_export`：允许生成和下载完整Excel。
- 用户只能访问本人所属 `hospital_id` 的运行和文件。
- 公司账号和无医院范围账号默认不能查看患者明细。
- 文件下载必须再次校验令牌、医院范围、权限和过期时间，不能仅凭文件路径访问。

## 脱敏

页面预览根据字段语义脱敏：

- 患者标识、住院号、病历号：保留前后少量字符，中间使用 `*`。
- 姓名：仅保留姓，其余使用 `*`。
- 手机号、证件号：只保留末四位。
- 时间、科室和本指标计算证据字段保持可读。

接口只返回脱敏预览。完整数据只进入受保护的短期快照和授权下载的Excel，不写入聊天消息、浏览器日志、执行链路或审计详情。

## 存储与清理

新增明细快照表和导出记录表，保存相对路径、文件哈希、行数、状态、创建时间和到期时间，不把文件内容写入数据库。清理任务在以下时机执行：

1. 应用启动时。
2. 创建或列出导出文件时。
3. 现有调度器每小时执行一次。

到期后删除文件并将导出记录标记为 `expired`。删除失败记录错误，下一次清理继续重试。

## 审计

以下动作写入数据访问审计表：

- 登录成功、失败和退出
- 查看统计范围、分子或未达标预览
- 创建Excel
- 下载Excel
- 权限拒绝
- 数量不一致导致导出停止
- 文件过期清理

审计记录包含用户、医院、指标、运行ID、导出ID、动作、行数、结果、请求ID、时间和失败原因，不包含患者字段值。

## 接口

计划增加：

| 接口 | 用途 |
| --- | --- |
| `POST /api/auth/hospital/login` | 医院账号登录 |
| `POST /api/auth/hospital/change-password` | 首次登录修改密码 |
| `POST /api/auth/hospital/logout` | 撤销会话 |
| `POST /api/sql-runs/{run_id}/details` | 创建或复用短期明细快照 |
| `GET /api/sql-runs/{run_id}/details/{group}` | 分页读取脱敏预览 |
| `POST /api/sql-runs/{run_id}/exports` | 生成Excel |
| `GET /api/indicator-exports` | 查看本人所属医院的导出记录 |
| `GET /api/indicator-exports/{export_id}/download` | 下载未过期Excel |

`group`只允许 `denominator`、`numerator`、`unmatched`。

## 失败处理

- 未登录：提示重新登录，不丢失当前聊天内容。
- 无查看权限：提示联系管理员开通“指标明细查看”。
- 无导出权限：允许预览，但禁用导出按钮并说明原因。
- 字段映射不完整：显示缺失的业务字段，不执行猜测查询。
- 数量不一致：停止生成并引导重新试运行。
- 超过20,000条：提示缩小统计区间。
- 文件已过期：提示重新生成，不返回服务器路径。
- DBHub或业务库失败：保留当前页面和筛选状态，显示可重试信息。

## 验收标准

1. 未登录或无权限用户不能调用明细和下载接口。
2. `user_001`首次使用 `123456` 登录后必须修改密码，修改后可查看和导出 `hospital_001` 数据。
3. 急会诊试运行显示576、488、88时，三个明细标签和Excel工作表数量一致。
4. 页面预览患者标识已脱敏，Excel中的授权完整值正确。
5. 文件生成于 `runtime/exports/hospital_001/{run_id}/`，Git状态不出现导出文件。
6. 下载、预览、权限拒绝和过期清理都有不含患者值的审计记录。
7. 业务数据变化造成数量不一致时不生成Excel。
8. 24小时后文件不可下载，清理任务删除磁盘文件并更新状态。
9. 明细SQL只包含配置的证据字段并通过只读安全校验。
10. 原有聚合试运行、聊天、执行链路和四个指标计算结果不受影响。
