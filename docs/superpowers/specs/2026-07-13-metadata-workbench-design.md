# 数据库与元数据工作台设计

## 1. 目标

将现有“DBHub MCP 测试”技术弹窗升级为正式的“数据库与元数据”业务工作台，使医院医务、质控人员和实施人员无需理解 DBHub、MCP 或原始接口即可完成数据库结构同步，并快速判断数据库结构变化是否影响指标运行。

本次只调整元数据管理入口、展示方式和查询接口，不改变现有只读同步边界，不同步患者业务数据，也不允许页面执行写入医院业务库的 SQL。

## 2. 页面入口与用户

- 在左侧“业务工作台”增加“数据库与元数据”，与“指标运算监控”同级。
- 移除“现有工具”中的元数据弹窗入口和“DBHub MCP 测试”名称。
- 当前医院沿用顶部医院 ID，页面切换或医院 ID 改变后重新读取该医院的元数据概况。
- 医院人员可以查看状态并手动同步已配置的医院业务库；系统管理库只在“连接详情”中展示，不提供主同步按钮。

## 3. 页面结构

页面采用安静、紧凑的运维工作台布局，不使用营销式卡片堆叠。

1. **页面标题栏**：显示“数据库与元数据”、当前医院、最近同步状态和“同步数据库结构”主按钮。
2. **状态概览带**：显示业务数据源、最近同步时间、表数量、字段数量。没有同步记录时明确引导首次同步。
3. **结构变化区**：按新增、删除、类型变化、可空性变化展示最近一次同步差异；无变化时显示“数据库结构与上次一致”。
4. **受影响指标区**：展示变化命中的指标编码、业务字段和数据库字段；无命中时说明本次变化未影响已配置指标。
5. **连接详情**：默认折叠，展示医院业务库和系统管理库的连接状态、只读能力和底层工具名称，供实施排障使用。

主界面使用医院人员能理解的中文。`DBHub`、`MCP`、工具名和连接地址只出现在连接详情中。

## 4. 数据流与接口

### 4.1 页面加载

新增 `GET /api/metadata/overview?hospital_id=<医院ID>`，返回：

- 当前医院最近一次同步的业务库和批次；
- 最近同步时间、表数量和字段数量；
- 最近批次的结构变化；
- 根据医院字段映射计算出的受影响指标；
- 是否存在可展示的同步记录。

页面同时复用 `GET /api/mcp/dbhub/sources` 获取当前可用数据源，但前端只把医院业务库作为主数据源，将 `wiki_agent_runtime` 归类为系统管理库并放入连接详情。

### 4.2 手动同步

继续复用 `POST /api/metadata/sync`。请求固定携带当前医院、用户选择的医院业务库和 `source=dbhub`。同步过程中禁用按钮，成功后重新读取概览；失败时保留当前内容并给出可执行的排查建议。

### 4.3 数据安全

- 同步只读取 `INFORMATION_SCHEMA.TABLES` 和 `INFORMATION_SCHEMA.COLUMNS`。
- 页面不读取或展示患者记录。
- 页面不展示数据库密码。
- 系统管理库不作为医院业务元数据的默认同步目标。

## 5. 状态与错误处理

- **加载中**：说明正在读取数据库结构状态，不以空白页面代替。
- **未同步**：显示业务数据源已连接，并引导点击“同步数据库结构”。
- **同步中**：按钮显示处理中并防止重复提交。
- **同步成功**：显示完成时间、批次摘要、变化数量和受影响指标数量。
- **连接失败**：区分数据源服务不可用与业务数据库不可访问，并提供“重新检查连接”。
- **同步失败**：说明本次未更新元数据，保留上一次成功结果并允许重试。
- **无结构变化**：明确显示结构与上次一致，不使用空列表表达。

## 6. 实现边界

- 延续现有单页应用和 `web/index.html`，新增 `metadata` 工作台路由，不引入新前端框架。
- 后端概览查询放入元数据模块，通过清晰的返回结构供 API 使用，避免在路由中拼接数据库查询。
- 复用现有元数据同步、快照差异和受影响指标计算逻辑。
- 本批不实现自动定时同步、数据库连接配置编辑、表字段全文浏览和字段映射编辑，这些作为后续增强。

## 7. 测试与验收

### 7.1 自动化测试

- 概览接口在无同步记录时返回稳定的空状态。
- 概览接口能返回最近批次、表字段数量、变化和受影响指标。
- 页面包含正式工作台入口，不再显示“DBHub MCP 测试”。
- 页面同步操作调用现有同步接口，成功后刷新概览。
- 系统管理库不会成为主同步选项。

### 7.2 浏览器验收

1. 登录后从左侧进入“数据库与元数据”。
2. 页面默认显示 `hospital_001` 和医院业务库状态。
3. 点击“同步数据库结构”，看到同步中状态且不能重复点击。
4. 成功后看到最近同步时间、4 张表、33 个字段以及本次变化摘要。
5. 再次同步且结构未变化时看到“数据库结构与上次一致”。
6. 展开“连接详情”后才能看到 DBHub、MCP 工具名和系统管理库。
7. 在桌面和移动宽度下检查文字、按钮和状态区域没有重叠或溢出。
