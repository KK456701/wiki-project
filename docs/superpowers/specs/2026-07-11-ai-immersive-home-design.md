# AI 沉浸式首页设计

## 目标

让 AI 对话在桌面端成为真正的主界面：移除重复品牌和说明区域，把对话工作区从当前约 `190px` 的纵向起点提升到窗口顶部约 `12px`，同时保留医院上下文、用户身份和系统运维入口。

沉浸式布局只用于 `#/assistant`。进入指标运算监控等专业业务页面时，恢复完整顶栏和侧栏。

## 已确认方向

采用方案 C“沉浸式 AI 模式”：

- AI 首页隐藏传统品牌顶栏。
- 左侧导航压缩为约 `64px` 的工具轨道。
- 医院、当前用户和系统工具变为右上角紧凑浮动控制组。
- 品牌名称合并到聊天工作区的当前会话栏。
- 删除 AI 首页独立的“今天需要处理哪个指标？”标题区域；该引导由欢迎消息承担。
- 对话工作区距离窗口上、右、下边缘约 `12px`，占满其余空间。
- 业务页面继续使用完整工作台布局。
- 移动端继续使用现有横向导航，不采用过窄的工具轨道。

## 信息架构

### 桌面 AI 首页

```text
┌──────┬────────────────────────────────────────────────────┐
│ AI   │ 核心制度指标 Agent · 当前会话       医院 / 用户 / 工具 │
│ 监   ├────────────────────────────────────────────────────┤
│ 稿   │                                                    │
│ 审   │                    对话消息                         │
│ 库   │                                                    │
│      ├────────────────────────────────────────────────────┤
│      │                    输入区域                         │
└──────┴────────────────────────────────────────────────────┘
```

左侧工具轨道负责页面与兼容工具入口；顶部会话栏负责品牌、会话和新会话操作；右上角浮动控制组负责医院和系统上下文。

### 专业业务页面

保持当前结构：

- 完整品牌顶栏
- 完整文字侧栏
- 业务主内容区
- 可打开的 AI 辅助抽屉

切换布局只改变 CSS class 和可见样式，不重建工作台、聊天或业务页面。

## 组件行为

### 工作台模式 class

`showAssistantPage()` 为 `#workbenchShell` 增加 `assistant-immersive`。

`prepareBusinessPage()` 移除 `assistant-immersive`。

该 class 是沉浸布局的唯一开关，不根据 URL 在 CSS 中重复推断状态。

### 顶栏复用

不得复制医院输入、用户身份或系统工具 DOM。AI 首页中继续使用现有 `.topbar-actions`：

- `.workbench-topbar` 在桌面沉浸模式使用 `display: contents`。
- `.brand` 隐藏。
- `.topbar-actions` 固定到右上角。
- 进入业务页后恢复普通顶栏布局。

这样医院 ID 输入和现有事件监听仍只有一套。

### 导航工具轨道

桌面沉浸模式下：

- 侧栏宽度约 `64px`。
- 隐藏分组标题和导航长名称。
- AI、监控导航显示已有短标识。
- 指标设计稿、审批、数据库与元数据增加 `data-short` 和 `title`，显示“稿 / 审 / 库”。
- 鼠标悬停和键盘聚焦时显示完整名称提示。
- 每个目标保持至少 `44px` 高，提供清晰焦点状态。

进入业务页后，原完整文字恢复。

### 会话栏

首页会话栏显示：

- “核心制度指标 Agent”
- 当前 session ID
- “新会话”按钮

紧凑抽屉中继续显示当前会话和新会话，不复制按钮。

### 欢迎态

删除 AI 页面独立标题栏后，欢迎消息承担首次引导：

- 产品身份
- 可执行任务
- 示例问题

已有消息时不再显示额外页面说明，释放全部纵向空间。

## 视觉系统

沿用现有医疗工作台色彩：

- 墨黑绿 `#172320`：工具轨道和主文字
- 医疗青 `#087b75`：当前入口、主操作和焦点
- 浅青 `#dff4f1`：选中与上下文提示
- 冷白 `#ffffff`：对话和控制面
- 雾灰绿 `#eef3f1`：页面背景
- 状态红 `#a64536`：异常和必要提示

字体继续使用系统中文无衬线字体处理业务正文，`SFMono-Regular / Consolas` 只用于 session ID、指标编码和技术数据。

本布局的识别特征是“深色医疗工具轨道 + 大面积白色对话工作面”，不增加渐变球、营销标题或装饰卡片。

## 响应式规则

### 大于 `760px`

- 应用沉浸式工具轨道。
- 对话工作区与窗口保持约 `12px` 间距。
- 浮动上下文控件固定在右上角。
- 消息区在会话栏和输入区之间独立滚动。

### `760px` 及以下

- 禁用沉浸式工具轨道规则。
- 恢复现有移动端顶栏和横向导航。
- 医院、用户和系统工具保持横向可滚动。
- 对话工作区占满剩余高度，AI 抽屉保持全屏。

## 状态与错误处理

- 路由切换不得清空 AI 消息、输入草稿、session ID 或监控选中状态。
- 医院 ID 仍由唯一 `#hospitalId` 输入管理。
- 管理员权限失效时保留 `#/monitoring`，重新登录后恢复业务页。
- 沉浸模式样式加载失败不会影响路由和聊天请求；页面仍保留语义化 HTML。
- 浮动控制组不得遮挡“新会话”或消息滚动条。

## 验收标准

1. 桌面端 `#/assistant` 的传统品牌顶栏不可见。
2. 对话工作区顶部距离窗口约 `12px`，不再经过独立 AI 标题区域。
3. 工具轨道约 `64px`，入口可点击、可聚焦并有完整名称提示。
4. 医院、用户和系统工具仍使用原 DOM 且可正常操作。
5. 当前会话栏显示产品名称、session ID 和新会话。
6. 切换到 `#/monitoring` 后完整顶栏和侧栏恢复。
7. 返回 AI 首页后对话和输入状态不丢失。
8. `390x844` 下继续使用横向导航，无横向页面溢出或过小触控目标。
9. 工作台、监控、聊天 SSE 和全量测试通过。

## 实施边界

本次修改限定在 `web/index.html`、`web/workbench.css`、`web/workbench.js` 和前端结构测试。不会修改聊天 SSE、会话记忆、Trace、监控 API、权限接口或后端 Agent 编排。
