# 患者入院 48 小时内转科指标统一口径设计

## 目标

将 `MQSI2025_001` 的时间起点、48 小时边界、有效转科选择和 ICU 排除规则固化为可审计的结构化口径，并保证汇总 SQL、分母明细和分子明细由同一份统计范围生成。

## 本批范围

- 仅适配医院 `hospital_001` 的公司 SQL Server 测试库 `WIN60_QA_991827`。
- 不修改医院业务库表结构，不创建视图，不写入患者业务数据。
- 保留 MySQL Demo 映射，正式医院映射通过规则导入流程写入运行库。
- 不在本批扩展通用计算 DSL 的多表关联能力，使用与急会诊相同的专用查询配置模式。

## 口径决策

| 口径项 | 本批采用值 | 结构化表达 |
|---|---|---|
| 统计对象 | 统计期间内办理住院的有效住院人次 | `INPATIENT_ENCOUNTER.ENCOUNTER_ID` 去重 |
| 统计时间 | 办理住院时间 | `INPATIENT_ENCOUNTER.ADMITTED_AT`，采用 `[start_time, end_time)` |
| 及时边界 | 转科发生在入院后 0 至 48 小时内，包含 48 小时整 | `0 <= DATEDIFF(MINUTE) <= 2880` |
| 有效转科 | 转科类型为跨科转科，或病区转移且来源科室与目标科室不同 | 类型代码与科室变化组合判断 |
| 多次转科 | 每次住院只取最早一条有效转科记录 | 按 `INPAT_TRANSFER_AT, INPAT_TRANSFER_ID` 排序取 `ROW_NUMBER() = 1` |
| ICU 排除 | 排除来源或目标科室/病区属于本院 ICU 机构的转科 | 使用医院级 `icu_org_ids` 参数，不使用公司底稿中的外院硬编码 |
| 异常时间 | 转科早于入院不计入分子 | 时间差下限为 0 |

上述决策是本院当前生效口径。国标定义本身不写入物理字段名；医院字段映射负责将“办理住院时间”“实际转科时间”等业务概念映射到本院数据库。

## 数据映射

| 业务概念 | 医院表字段 |
|---|---|
| 医院 | `WINDBA.INPATIENT_ENCOUNTER.HOSPITAL_SOID` |
| 住院人次 | `WINDBA.INPATIENT_ENCOUNTER.ENCOUNTER_ID` |
| 办理住院时间 | `WINDBA.INPATIENT_ENCOUNTER.ADMITTED_AT` |
| 转科时间 | `WINDBA.INPAT_TRANSFER.INPAT_TRANSFER_AT` |
| 转科类型 | `WINDBA.INPAT_TRANSFER.INPAT_TRANSFER_TYPE_CODE` |
| 转出科室/病区 | `ORIGIN_DEPT_ID` / `ORIGIN_WARD_ID` |
| 转入科室/病区 | `DESTINATION_DEPT_ID` / `DESTINATION_WARD_ID` |
| 数据有效状态 | 两表的 `IS_DEL` |

## 查询结构

新增 `inpatient_transfer_48h_sqlserver` 查询配置。汇总和明细共同使用以下逻辑：

1. `eligible_encounter`：按医院、有效状态、住院业务类型和半开统计区间筛选分母住院人次。
2. `valid_transfer`：按有效转科类型和 ICU 排除规则筛选转科，并按实际转科时间排序。
3. `base`：每个住院人次左连接最早一条有效转科，计算分钟差。
4. 汇总查询从 `base` 计算分母、分子、未达到要求数量和比例。
5. 明细查询直接输出同一个 `base`，用 `__meets_numerator` 标记分子记录。

明细快照沿用现有一致性保护：生成后重新计算明细分子和分母，必须与试运行日志中的汇总数量完全相等，否则拒绝生成明细并提示重新试运行。

## 配置与发布

- 正式导入时写入 `MQSI2025_001` 的字段映射、两表关联关系、医院参数和 SQL Server 专用 SQL。
- 必填医院参数包括医院范围值、两类转科代码和 ICU 机构 ID 列表。
- 任一必填参数为空时，医院映射保持“待确认”，不得生成可执行 SQL。
- 当前 QA 库核验到的 ICU 机构仅作为本地配置，不提交真实连接密码或患者数据。

## 前端表现

- 用户询问该指标后可以正常生成 SQL 和试运行。
- 结果说明展示数据库、两张来源表、起算字段、边界、最早转科选择和 ICU 排除规则。
- 点击分母、分子或未达到要求的详情时，使用同一快照分页和导出。
- 医生看到业务解释；SQL、字段名和查询配置仍放在“技术详情”中。

## 验收标准

1. SQL Server 导入后，`MQSI2025_001` 映射状态为已确认，且不再引用 Demo 表。
2. 汇总 SQL 和明细 SQL 同时包含相同的医院、时间、有效状态、转科类型、最早事件和 ICU 排除条件。
3. 48 小时边界采用分钟级闭区间，负时间差不进入分子。
4. 对同一运行记录，明细分母数等于汇总分母数，明细分子数等于汇总分子数。
5. 现有急会诊及 MySQL Demo 指标测试不回归。
6. 使用当前 QA 数据、统计区间 `2026-06-01` 至 `2026-08-01` 时，可只读复现候选结果：分母 158、分子 2、比例 1.27%。该数字只用于技术验收，不写死到代码。
