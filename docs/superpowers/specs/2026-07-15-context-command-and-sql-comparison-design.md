# 会话参数命令与 SQL 差异报告设计

## 目标

修复两个用户可见问题：

1. 用户修改统计时间后，系统只确认新的统计区间，并把它保存为后续生成 SQL、试运行和诊断使用的会话状态，不再重复指标定义和公式。
2. 用户粘贴 SQL 进行诊断时，主报告固定比较“用户 SQL”和“当前生效 SQL”；国标口径只用于解释当前生效口径的来源，不参与主结论争夺。

## 设计

### 统计时间更新

沿用现有日期解析和结构化会话状态，不新增意图类型。`query` 请求在成功解析并保存统计区间后，直接返回确定性确认文案，跳过普通 LLM 指标回答。确认文案必须包含起止时间、“结束时刻不计入”和后续影响范围。

后续“生成 SQL”“试运行”“诊断”继续从结构化会话状态读取该区间，模型原始对话仅用于语言理解，不作为执行参数来源。

### SQL 差异诊断

程序生成两张确定性表格：

- 试运行结果：用户 SQL、当前生效 SQL的执行状态、分子、分母和指标值。
- 计算规则差异：比较项目、用户 SQL、当前生效 SQL、对结果的影响和建议。

当前生效 SQL 使用医院已审批且生效的计算口径；医院没有有效定制时，它等同于国标口径。国标独立试运行状态只放在折叠的技术依据中，不能被描述为“用户 SQL 的对比对象”，其执行失败也不能覆盖用户 SQL 与当前生效 SQL的有效比较结果。

### 模型职责

程序负责事实、标签、数值和表格；本地模型只负责把已经核验的事实组织成通俗说明。模型输出未包含正确比较对象或引入未经核验事实时，使用程序生成的稳定回答。

## 安全边界

- 继续只允许单条只读查询试运行。
- 报告只展示聚合结果，不展示患者明细或原始 SQL。
- 不因国标参考 SQL 执行失败而把数据库故障作为主结论。

## 验收标准

1. 输入“统计时间改为2026-06-01至2026-08-01”后，回复确认新时间，不出现指标定义或公式。
2. 紧接着输入“生成 SQL”，生成结果使用 `2026-06-01 00:00:00` 至 `2026-08-01 00:00:00`。
3. 粘贴 SQL 诊断报告的主比较表只出现“用户 SQL”和“当前生效 SQL”。
4. 报告包含逐项差异表；国标信息只出现在口径来源或折叠技术详情中。
5. 用户 SQL 与当前生效 SQL试运行成功时，即使国标参考 SQL 失败，主结论仍基于这两段 SQL 的差异。
